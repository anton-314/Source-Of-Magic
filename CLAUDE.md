# CLAUDE.md — AI Guidance for Source of Magic

This file provides instructions for Claude and all other AI assistants working in this repository.

---

## Project Overview

**Source of Magic** is an open-source, community-driven epic fantasy novel managed through Git.
It treats worldbuilding as documentation, characters as modules, and the plot as source code.
The working language of this project is **English** — all files, commit messages, issues, PRs,
and AI responses must be in English.

---

## AI Role and Boundaries

### What AI must NOT do

- Write, edit, rewrite, or suggest changes to prose in `manuscript/`
- Author or alter lore, character sheets, or worldbuilding documents in `world/`
- Make creative decisions about plot, characters, magic, or geography
- Open issues or PRs that propose story or lore changes

The story and its creative direction are exclusively the human author's domain.

### What AI is responsible for

- Managing and explaining the **Git workflow** (branching, commits, merges, tags)
- Maintaining and evolving the **project structure** (directories, naming conventions, CI files)
- Writing, debugging, and documenting **scripts** in `scripts/`
- Configuring **GitHub Actions workflows** in `.github/workflows/`
- Keeping **issue templates**, **PR templates**, and **markdownlint config** up to date
- Running and interpreting script output on behalf of the author (e.g., word counts, lore checks)

### What AI may do (read-only creative access)

AI assistants may read all chapters in `manuscript/` and all documents in `world/` in order to:

- Answer questions about the story, plot, characters, or world
- Cross-reference lore consistency when asked
- Help the author recall details from earlier chapters

This access is read-only. Reading is allowed; writing is not.

## Scripts Reference

All scripts require **Python 3.10+** and are run from the repository root.
Export scripts additionally require [Pandoc](https://pandoc.org/installing.html);
`export_pdf.py` also requires XeLaTeX (`texlive-xetex`).

| Script | Purpose | Command |
|---|---|---|
| `new_chapter.py` | Creates the next `chapter-NN.md` with correct numbering | `python3 scripts/new_chapter.py` |
| `new_chapter.py` | Same, with a chapter title | `python3 scripts/new_chapter.py "The Storm Breaks"` |
| `word_counter.py` | Per-chapter word count and grand total | `python3 scripts/word_counter.py` |
| `validate_lore.py` | Checks broken links, empty world sections, placeholder chapters | `python3 scripts/validate_lore.py` |
| `character_index.py` | Lists which chapters mention each character from `world/characters/` | `python3 scripts/character_index.py` |
| `export_md.py` | Concatenates all chapters into `dist/source-of-magic.md` | `python3 scripts/export_md.py` |
| `export_pdf.py` | Builds `dist/source-of-magic.pdf` via Pandoc + XeLaTeX | `python3 scripts/export_pdf.py` |
| `export_epub.py` | Builds `dist/source-of-magic.epub` via Pandoc | `python3 scripts/export_epub.py` |

All export scripts accept `--output <path>` to override the default output location.

## CI/CD Pipelines

Both workflows run on every push to `main` and can be triggered manually via
`workflow_dispatch`.

### `lint.yml` — Markdown Linting

Runs `markdownlint-cli2` against all `*.md` files using the rules in `.markdownlint.json`.

Active rule overrides:

| Rule | Status | Reason |
|---|---|---|
| MD013 (line length) | disabled | Long prose lines are acceptable |
| MD036 (emphasis as heading) | disabled | Style choice in narrative text |
| MD041 (first line heading) | disabled | Chapter files don't start with H1 document title |

Linting runs on every push and every PR. Fix lint errors before merging.

### `build-book.yml` — Book Compilation

Concatenates all chapters in order, then builds:

- `dist/source-of-magic.pdf` via Pandoc + XeLaTeX
- `dist/source-of-magic.epub` via Pandoc

The artifact `source-of-magic-book` is uploaded to GitHub Actions after each successful build.

## Naming Conventions

- **Chapters:** `chapter-NN.md` — zero-padded two-digit number (e.g., `chapter-01.md`, `chapter-12.md`)
- **Character files:** `firstname-lastname.md` in `world/characters/` (e.g., `aria-voss.md`)
  — `character_index.py` derives the display name from the filename stem
- **Geography/lore files:** lowercase kebab-case `.md` files inside their respective `world/` subdirectory
- **Branches:** use descriptive names, e.g., `chapter-04-draft`, `fix/lore-broken-links`, `feat/character-aria-voss`
- **Commit messages:** imperative mood, English, concise (e.g., `Add chapter 04 draft`, `Fix broken link in geography`)

## Git Workflow

1. Work on a feature branch, never directly on `main`
2. Open a PR using the provided template — fill out all checklist items
3. CI must pass (lint + build) before merging
4. Squash or rebase before merging to keep history clean
5. Delete the branch after merging

The `dist/` directory is not committed — it is produced by CI and local script runs only.

## Adding Content

### New chapter

```bash
python3 scripts/new_chapter.py "Optional Title"
```

This creates the correctly numbered file. Write prose inside it; do not rename the file.

### New character

Create `world/characters/firstname-lastname.md`. Include: background, motivation, abilities,
and story arc. The character then appears automatically in `character_index.py` output.

### New geography or lore document

Add a `.md` file to `world/geography/` or `world/magic-system/`. There is no generator
script — create the file directly. Keep filenames lowercase kebab-case.

## Common AI Tasks

These are recurring tasks AI is expected to handle without needing extra instructions:

- **Run word count:** `python3 scripts/word_counter.py` and report the output
- **Run lore validation:** `python3 scripts/validate_lore.py` and explain any warnings
- **Run character index:** `python3 scripts/character_index.py` and summarize which characters appear where
- **Create a new chapter file:** use `new_chapter.py`, confirm the filename created
- **Check CI status:** use `gh run list --limit 5` to show recent workflow runs
- **Explain project structure:** refer to the structure section in this file
- **Answer questions about the story or world:** read the relevant files in `manuscript/` and `world/`, then answer — do not invent details not present in those files
